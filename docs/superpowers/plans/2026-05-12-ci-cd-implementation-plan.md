## `.github/workflows/release.yml` — Full Implementation

```yaml
name: Release CI/CD

on:
  push:
    tags: ['v*']
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

env:
  CARGO_TERM_COLOR: always
  RUST_BACKTRACE: 1

permissions:
  contents: write  # Required for writing release assets and tags
  actions: read    # Read workflow run info

jobs:
  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version-file: '.nvmrc'
          cache: 'pnpm'
          cache-dependency-path: 'pnpm-lock.yaml'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm lint

      - name: Typecheck
        run: pnpm typecheck

  build:
    name: Build (${{ matrix.platform }})
    needs: lint
    strategy:
      fail-fast: false
      matrix:
        platform: [ubuntu-latest, windows-latest, macos-latest]
    runs-on: ${{ matrix.platform }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version-file: '.nvmrc'
          cache: 'pnpm'
          cache-dependency-path: 'pnpm-lock.yaml'

      - name: Setup Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          toolchain-file: 'rust-toolchain.toml'

      - name: Cache cargo registry
        uses: actions/cache@v4
        with:
          path: ~/.cargo/registry/index
          key: cargo-registry-${{ hashFiles('src-tauri/Cargo.lock') }}
          restore-keys: cargo-registry-

      - name: Cache cargo target dir
        uses: actions/cache@v4
        with:
          path: src-tauri/target
          key: cargo-target-${{ matrix.platform }}-${{ hashFiles('src-tauri/Cargo.lock') }}
          restore-keys: cargo-target-${{ matrix.platform }}-

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build Tauri app
        run: pnpm tauri build
        timeout-minutes: 60

      - name: Upload Release Asset
        if: startsWith(github.ref, 'refs/tags/v')
        uses: softprops/action-gh-release@v2
        with:
          files: |
            src-tauri/target/release/bundle/**/*.msi
            src-tauri/target/release/bundle/**/*.deb
            src-tauri/target/release/bundle/**/*.app.tar.gz
            src-tauri/target/release/bundle/**/*.dmg
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # Optional: Build summary for visibility
  build-summary:
    name: Build Summary
    needs: [lint, build]
    runs-on: ubuntu-latest
    if: always()
    steps:
      - name: Summary
        run: |
          echo "## Build Results" >> $GITHUB_STEP_SUMMARY
          echo "| Platform | Status |" >> $GITHUB_STEP_SUMMARY
          echo "|----------|--------|" >> $GITHUB_STEP_SUMMARY
          echo "| Lint | ${{ needs.lint.result }} |" >> $GITHUB_STEP_SUMMARY
          echo "| Ubuntu | ${{ needs.build.result }} |" >> $GITHUB_STEP_SUMMARY
          echo "| Windows | ${{ needs.build.result }} |" >> $GITHUB_STEP_SUMMARY
          echo "| macOS | ${{ needs.build.result }} |" >> $GITHUB_STEP_SUMMARY
```

**Note on build summary:** Matrix job results are not directly accessible via outputs. We use `needs.<job>.result` which shows pass/fail status for each platform build in the matrix. The `softprops/action-gh-release` automatically uploads matching files from `src-tauri/target/release/bundle/`.

## Test Plan

Since this is a CI/CD infrastructure change (no application code changed), traditional tests don't apply.

**Verification steps:**
1. Push a test tag: `git tag v0.0.0-test && git push origin v0.0.0-test`
2. Watch GitHub Actions tab — all 3 platform builds should start
3. After ~10-30 min, check Releases page — all 3 installers should appear
4. Download each installer and verify it launches

**Smoke test for built artifacts:**
- Windows: run `.msi` — app launches, window appears
- Linux: `sudo dpkg -i .deb` — app installs and launches
- macOS: extract `.app.tar.gz` — app runs (may show Gatekeeper warning, expected for unsigned)

## Failure Modes

| Scenario | How it manifests | Mitigation |
|----------|------------------|------------|
| Wrong bundle targets | CI succeeds but only one platform has artifacts | Fixed by D1 decision |
| Missing typecheck script | CI lint job fails with "command not found" | Fixed by D2 decision |

| Cache miss on first run | Build takes 20-30 min instead of 10-15 | Expected — first run always slower |
| Release upload fails | Assets not attached to GitHub Release | Check `softprops/action-gh-release` version |
| macOS notarization | Gatekeeper blocks .app | Expected for open-source, out of scope |
